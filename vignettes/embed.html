Output as HTML

Among other things fansi provides the sgr_to_html function which converts the ANSI CSI SGR sequences into HTML markup:

fansi::sgr_to_html(sgr.string)
## [1] "<span style='color: #0000BB;background-color: #BBBB00;'>day > night</span>"
## [2] "<span style='color: #BBBB00;background-color: #0000BB;'>dawn < dusk</span>"
## [3] "boom boom"

This however is insufficient as knitr / rmarkdown / pandoc escape all the HTML markup so the browser does not interpret.

Knitr Integration

Thankfully Yihui Xie built a lot of flexibility into knitr so we can override the existing output handler

knitr allows us to set hooks that process the output of evaluated R code chunks:

knitr::knit_hooks$set(output = FUN)

FUN is a function that takes the raw screen output (captured as text) produced by the R code chunk and modifies it for further processing by pandoc into the final output.

In our case we prefer to only handle output that contains ANSI CSI SGR, and let knitr do its thing the rest of the time. The best way we have found to do this is unfortunately a little awkward:

old.hook <- knitr::knit_hooks$get("output")
new.hook <- function(x, options){

  # If the output has SGR in it, then:
  if(any(fansi::has_sgr(x))) {
    code_block(                  # add <PRE> & <CODE> HTML tags
      fansi::sgr_to_html(        # convert SGR to HTML
        x = html_esc(x)          # escape HTML special chars in input
    ) )
  }
  # If output doesn't have SGR, then:
  else {
    old.hook(x, options)         # use the default output hook
  }
}
knitr::knit_hooks$set(output = new.hook)

The helper functions html_esc and code_block are defined in the appendix.

Now, we get the desired output:

writeLines(sgr.string)
## day > night
## dawn < dusk
## boom boom

Appendix

These are the helper functions we used in our knit hook. A couple of notes;

  • You should use a proper HTML escape function such as htmltools::htmlEscape; we do not do so here to avoid an additional dependency.
  • As far as we can tell pandoc when outputting chunk output uses unclassed <PRE> and <CODE> HTML tags. In the future this might change, which may cause code output with SGR in it to be formatted differently than code output without it.
## Escape HTML entities in raw *input* so they are not interpreted by the
## browser

html_esc <- function(x) gsub(">", "&gt;", gsub("<", "&lt;", x))

## Wrap the *output* in <code> and <pre> tags.

code_block <- function(x)
  sprintf(
    "<pre class=\"r-fansi-output\"><code>%s</code></pre>",
    paste0(x, collapse="\n")
  )
